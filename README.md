# Редактор графов для mk-eda

> Код самого редактора графов находится по пути `mk_eda/tools/graph_editor/main.py`.

> Этот инструмент является частью проекта **[mk-eda](https://mks2.cs.msu.ru/prac/mk-eda)**.  
> Он предоставляет визуальный интерфейс для создания, редактирования ориентированных ациклических графов, используемых в логическом синтезе и верификации (Inverter Graphs, включая AIG-подобные структуры).

## Возможности

- **Создание и редактирование вершин**  
  - Поддерживаемые типы вершин: `input`, `const`, `and`, `or`, `xor`, `majority`.  
  - Вершины можно помечать как **выходные** (отображаются с обводкой).  
  - Каждая вершина показывает своё имя и тип. Двойной щелчок по вершине выводит её глубину в консоль (в область лога).

- **Рисование рёбер (положительных и отрицательных)**  
  - Рёбра могут быть сплошными (положительная связь) или пунктирными (отрицательная, обозначает инверсию).  
  - Несколько параллельных рёбер между одной парой вершин автоматически разводятся для читаемости.  
  - Рёбра рисуются в виде дуг со стрелками.

- **Проверка корректности графа**  
  - Редактор непрерывно проверяет, соответствует ли граф правилам для каждого типа вершины (например, `and`/`or`/`xor` требуют ровно два входа, `majority` — три).  
  - Корректные компоненты связности подсвечиваются зелёным цветом; некорректные — красным.
  - Проверка происходит автоматически после любого изменения.

- **Контекстные меню**  
  - ПКМ по вершине: удалить вершину, удалить выделенные вершины, показать/скрыть выражение, сменить статус выходной вершины, изменить свойства вершины.  
  - ПКМ по ребру: удалить ребро.  
  - ПКМ по свободному пространству: создать новую вершину, очистить сцену.

- **Предпросмотр логического выражения**  
  - Для выходных вершин можно отобразить соответствующее булево выражение прямо над вершиной. (Не обновляется при изменении графа и применяемо только к выходным вершинам)

- **Сохранение и загрузка**  
  - Сохранение текущего графа в JSON-файл (используется внутренний формат).
  - Загрузка ранее сохранённого графа из JSON-файла.

- **Операции над графом (экспериментальные)**  
  - *Оптимизация по сложности* (XAIG complexity optimizer).  
  - *Оптимизация по глубине* (XAIG depth optimizer).  
  > ⚠️ **Данные функции находятся в стадии эксперимента и могут работать нестабильно. Рекомендуется предварительно сохранять граф перед их применением.**

- **Инструменты навигации и редактирования**  
  - **Инструмент «Стрелка»** – выделение рамкой и перетаскивание вершин.  
  - **Инструмент «Рука»** – панорамирование вида.  
  - **Инструмент «Ластик»** – щелчок по вершине/ребру удаляет его.  
  - **Инструмент «Ребро»** – переключение между положительным (сплошным) и отрицательным (пунктирным) ребром; затем нужно щёлкнуть по двум вершинам, чтобы создать ребро.  
  - Масштабирование колёсиком мыши (плавное анимированное приближение/отдаление).

- **Горячие клавиши**  
  - `Delete` – удалить все выделенные элементы.  
  - `Ctrl+C` / `Ctrl+V` – скопировать и вставить выделенные вершины вместе с рёбрами между ними.  
  - `Пробел` – расширить выделение до всей компоненты связности (компонент).  
  - `Caps Lock` – переключение между инструментом «стрелка» и «рука».  
  - `Escape` – вывести в лог информацию о компонентах связности и текущий граф.

- **Панель лога**  
  - В нижней части окна отображается стандартный вывод и сообщения логгера из библиотеки `mk-eda`, что позволяет видеть результаты операций или ошибки.

## Требования

- **Python 3.9** (строго версии 3.9, работоспособность на других версиях не гарантируется)
- [Poetry](https://python-poetry.org/) для управления зависимостями и виртуальным окружением.

Все необходимые библиотеки, включая PySide6, будут установлены автоматически через Poetry.

## Установка и запуск

1. **Клонируйте репозиторий проекта** (если ещё не сделали этого):
   ```bash
   git clone https://github.com/llDmitryll/mk-eda_frontend.git
   cd mk-eda_frontend
   ```

2. **Настройте виртуальное окружение Poetry с Python 3.9**:
   ```bash
   poetry env use python3.9
   ```
   Убедитесь, что у вас установлен Python 3.9. Если команда не находит нужную версию, укажите полный путь до интерпретатора, например:
   ```bash
   poetry env use /usr/bin/python3.9
   ```

3. **Установите зависимости**:
   ```bash
   poetry install
   ```

4. **Запустите редактор графов из корневой папки проекта**:
   ```bash
   poetry run python mk_eda/tools/graph_editor/main.py
   ```

## Примеры использования
### Загрузка готового графа
В папке tests/verification находятся примеры графов в формате JSON. Вы можете загрузить любой из них через меню File → Open file и модифицировать его в редакторе.

### Создание графа с нуля
Щёлкните правой кнопкой мыши на пустом месте, выберите Create node, задайте имя и тип. Создайте несколько вершин, затем с помощью инструмента «Ребро» соедините их, формируя AIG-подобный граф. Помечайте вершины как выходные по необходимости.
После построения корректного графа можно опробовать экспериментальные оптимизации (см. выше).

### Сохранение результата
Экспортируйте полученный граф в JSON через File → Save graph to .json.
